# 배포 가이드 (라이브 URL)

`web/` 폴더는 **완전한 정적 사이트**(index.html + app.js + styles.css + data.json)라 어떤 정적 호스팅에도 바로 올라갑니다. 신청서의 "실제 서비스 URL"에 쓸 라이브 주소를 만듭니다. 가장 쉬운 순서로:

## 옵션 A — Netlify Drop (가장 쉬움, 로그인 불필요급, 1분)
1. https://app.netlify.com/drop 접속
2. **`margin-radar/web` 폴더를 통째로 드래그&드롭**
3. 즉시 `https://xxxx.netlify.app` URL 발급 → 끝
   - (계정 만들면 URL 고정·재배포 가능)

## 옵션 B — Vercel (CLI, 재배포 편함)
```bash
npm i -g vercel        # 최초 1회
cd margin-radar/web
vercel --prod          # 브라우저 로그인 후 배포 → URL 발급
```
- 데이터 갱신 후 재배포: `cd web && vercel --prod`

## 옵션 C — GitHub Pages (무료·영구)
1. GitHub 저장소 생성 → `web/` 내용을 push
2. Settings > Pages > Branch 지정 → `https://<id>.github.io/<repo>` 발급

## 자동 갱신과 연결 (선택)
매일 cron이 `web/data.json`을 갱신하므로:
- **Netlify/Vercel**: 갱신 후 재배포 명령을 `run_daily.sh` 끝에 추가하면 라이브도 매일 자동 갱신
  - 예: `vercel --prod --cwd ../web` 또는 Netlify CLI `netlify deploy --prod --dir=../web`
- **간단 버전**: data.json만 정적 호스팅에 매일 덮어쓰기(웹은 5분마다 자동 폴링하므로 새로고침 없이 반영)

## 제출용 권장
- **데모 안정성** 우선이면 옵션 A(Netlify Drop)로 고정 URL 하나 만들어 제출.
- 발표 시 "매일 자동 갱신"을 보여주려면 옵션 B/C + 자동 재배포 연결.

## 주의
- data.json에 API 키 없음(키는 `.env`·파이프라인에만 있고 웹엔 결과만) → 정적 배포 안전.
- `.env`, `.venv`, `data/`, `pipeline/`는 **배포 대상 아님**(웹은 `web/`만 올리면 됨).
