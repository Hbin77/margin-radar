# 수요(납품요구) 데이터 다운로드 안내

수요 축(정부가 실제로 얼마나 사는가)은 **공개 API로 안 나옵니다**(납품요구 API 404, 통계 API 빈 데이터, 허브 보고서 로그인 필요). 그래서 로그인 다운로드 1회가 필요합니다. 이 파일만 받아주시면 MR이 "마진 × 진입가능 × **수요** × 저경쟁"으로 완성됩니다.

## 받는 법 (5분)

1. **조달데이터허브 로그인**: https://data.g2b.go.kr → 우측 상단 로그인(나라장터 계정/공동인증서)
2. **보고서 열기**: 보고서목록 → "종합쇼핑몰 납품요구 물품 내역"(보고서ID UI-ADOXAA-038R) → **`21년 1월이후자료(조회속도향상)`** 클릭
3. **조회조건**:
   - **납품요구 결재일자**: 최근 1개월 (예: 2026-03-01 ~ 2026-03-31)  ※ 너무 길면 느림
   - (선택) 세부품명/물품분류로 우리 카테고리만 좁히면 더 빠름
4. **[조회]** → 결과 표 위/우측 **엑셀 다운로드**
5. 받은 파일을 **`margin-radar/data/dlvr_raw.csv`** 로 저장
   - 엑셀(.xlsx)로 받았으면 "다른 이름으로 저장 → CSV UTF-8"로 변환하거나, 그대로 주시면 변환해 드립니다.

## 그다음 (제가 자동 처리)
```
python3 pipeline/ingest_dlvr.py        # 품명별 수요금액·거래건수·수요기관수 집계 → demand.csv
python3 pipeline/score_v2.py           # 수요 축이 자동으로 점수에 반영(0.30마진+0.30진입+0.25수요+0.15저경쟁)
python3 pipeline/build_web_data.py
python3 pipeline/enrich_specdoc.py
```
→ 랭킹/상세에 "연간 정부 구매액·거래건수·수요기관수"가 추가되고, 블루오션 점수가 수요까지 반영한 완성형이 됩니다.

## 참고
- 한 달치 슬라이스면 데모로 충분합니다(연 환산/추세는 추후 여러 달 누적).
- 파일 헤더가 한글이어도 ingest가 자동 매칭합니다(납품금액/세부품명/수요기관).
