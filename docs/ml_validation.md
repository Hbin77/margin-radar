# ML 모델 적대적 검증 보고서

목표: "완벽한 모델인가" — 단일 holdout R²를 믿지 않고 다각도로 검증.
도구: `pipeline/ml_validate.py` (표본 12만행, 세부품명 932종)

## 검증 항목과 결과

### 1) 카테고리 룩업을 이기는가 (모델이 단순 룩업 이상인가)
| 방법 | R² | MAE |
|---|---|---|
| 전체 중앙값 | -0.03 | 7,620,085 |
| 세부품명별 중앙값(룩업) | 0.838 | 3,906,404 |
| **ML(HGB)** | **0.921** | **3,052,870** |

- 성능의 큰 부분(R² 0.84)은 "어느 품목이냐"가 결정. **ML은 그 위에 MAE를 약 22% 추가 개선** → 규격·제조사 등 카테고리 내 변동을 실제로 학습함(룩업 이상의 가치 확인).

### 2) 안정성 / 과적합
- 5-fold CV R² = **0.920 ± 0.002** (매우 안정)
- 과적합 갭(train−test R²) = **0.011** (거의 없음 — 건강)

### 3) 일반화 (처음 보는 세부품명, GroupKFold)
- 미지 세부품명 R² = **0.37** (계층 롤업 피처 추가로 0.33→0.37)
- 일반 R²(0.92)와 큰 차이 → **모델은 학습한 카테고리 내 보간에 강하고, 완전히 새로운 품목 가격 예측엔 약함.** (가격이 본질적으로 품목 특정적 → 근본적 한계)

### 4) 잔차 진단
- 잔차(log) 평균 ≈ 0 (편향 없음), 표준편차 0.64 → **통상 ~2배 오차**
- 예측오차 ±20% 이내 = **37%**, |빗나감|>170% = 약 10%

## 개선 후 재검증 (규격 파싱 강화 + 모델 튜닝)
- 규격에서 용량·무게·매수·인치·와트·세트·계약기간 등 추가 파싱 + HGB 튜닝 적용
- **HGB R² 0.918→0.940, 앙상블 0.948 / 룩업대비 MAE +22%→+31% / ±20%이내 37%→42% / 심한오차 10%→6.9% / 잔차std 0.648→0.558**
- 미지 카테고리 R² 0.33→0.39 (계층 롤업 피처). 근본 한계는 잔존.

## 종합 판정: "완벽하지 않다(원리상 불가) — 그러나 잘 검증되고 개선된 견고한 모델"

**강점**
- 통계적으로 건강: 안정적 CV, 과적합 없음, 편향 없는 잔차
- 단순 룩업을 의미 있게(MAE 22%) 능가 → 진짜 ML 가치

**한계 (정직 고지)**
- 가격의 대부분은 품목 종류가 결정(룩업 R² 0.84) — 모델의 순수 기여는 그 위의 부분
- 미지 카테고리 일반화 약함(R² 0.37)
- 개별 예측 정밀도 중간(±20% 이내 37%, 통상 ~2배 오차) → **"적정가"로 단정하기보다 "가격 수준 추정"**

## 최종 재검증 (2026-06-15, 신선 데이터 455,603행 / 표본 12만·세부품명 935종)
배포 파이프라인(abcd NAS 컨테이너)에서 동일 `ml_validate.py` 재실행 — 더 큰 데이터에서 모든 지표 유지/소폭 개선.

| 항목 | 결과 | 판정 |
|---|---|---|
| 전체중앙값 베이스라인 | R²=-0.03, MAE 7,301,781 | (무의미 — 기준점) |
| 세부품명 룩업 베이스라인 | R²=0.840, MAE 3,671,360 | 가격의 84%는 "품목 종류"가 결정 |
| **ML(HGB)** | **R²=0.941, MAE 2,445,650** | **룩업 대비 MAE +33.4%** → 규격·제조사 등 카테고리 내 변동 학습 입증 |
| 5-fold CV | R²=0.939 ± 0.002 | 매우 안정 |
| 과적합 갭(train−test) | 0.014 | 거의 없음(건강) |
| GroupKFold(미지 세부품명) | R²=0.411 ± 0.090 | 학습 카테고리 내 강함 / 신규 품목 약함(원리적 한계) |
| 잔차(log) | 평균 -0.008, std 0.544 | 편향 없음 |
| 예측오차 ±20% 이내 | 42.3% | 개별 예측은 "수준 추정"용 |
| 심한 오차(|잔차|>1.0) | 6.3% | — |

**누수 점검(코드 정적 검사)**: 타깃 `log(계약단가)`에 대해 피처는 규격 파싱·범주·제조사·계약기간만 사용, 가격 파생값(`orderCalclPrceAmt` 등) **미사용** → 타깃 누수 없음. 고차원 범주는 `TargetEncoder(CV)`로 학습 중 타깃-범주 누수 차단.

**알려진 경미한 흠(정직 고지)**: `과지출지표`는 전체데이터 in-sample 예측 기반(품목별 중앙값이라 영향 작음, 엄밀히는 out-of-fold가 정석). `makr_freq`/`sebu_freq`는 전체빈도(타깃누수 아님·무해).

## 올바른 사용 (한계 반영)
- 모델 출력은 **개별 정밀 가격이 아니라 상대 신호**로 사용:
  - 품목 단위 **중앙 잔차 = 과지출 신호**(개별 노이즈 평균화되어 더 안정)
  - **KMeans 시장 세분화 / IsolationForest 이상거래**(상대적 위치)
- 랭킹 점수는 ML에 의존하지 않음(마진·진입·수요·경쟁 투명 공식) → ML 한계가 랭킹을 오염시키지 않음.
- "완벽"은 불가: 동일 품목도 계약별 가격 편차(물량·시기·지역)가 있어 **환원 불가능한 분산** 존재.
