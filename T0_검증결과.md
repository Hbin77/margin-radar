# T0 데이터 피저빌리티 검증 결과 — GO

검증일: 2026-06-13 / 키: data.go.kr 개발계정 (자동승인, 2026-06-13~2028-06-13)

## 결론: GO (통과)
조달 마진 레이더의 핵심 축(단가·경쟁·품목매칭)을 깨끗한 OpenAPI로 확보. 엔진 로직이 실데이터에서 계산됨을 확인.

## 확인된 것

### 1. 종합쇼핑몰 품목정보 API — 완전 작동
- 엔드포인트: `https://apis.data.go.kr/1230000/at/ShoppingMallPrdctInfoService/getMASCntrctPrdctInfoList`
- 응답: **88개 컬럼**, 등록일 하루치 ≈ 26행
- 핵심 필드 확보:
  - 마진: `cntrctPrceAmt`(계약단가), `orderCalclPrceAmt`(주문산정가), `dscntAmt`(할인)
  - 경쟁/업체: `cntrctCorpNm`, `cntrctCorpNo`, `entrprsDivNm`(기업구분)
  - 품목매칭: `prdctClsfcNoNm`(품명), `dtilPrdctClsfcNo`(세부품명번호), `prdctIdntNo`(물품식별), `prdctSpecNm`(규격), `prdctMakrNm`(제조사), **`prdctImgUrl`(상품이미지)**
  - 정책 플래그: `exclncPrcrmntPrdctYn`(우수제품), `smetprCmptProdctYn`(중기경쟁제품)
- 오퍼레이션: MAS(다수공급자) / 일반단가 / 제3자단가 별도 존재

### 2. 엔진 로직 실증 (426행 표본, 30개 세부품명)
품목별로 [경쟁업체수 + 단가 min~max] 계산 성공. 예:
- 심장충격기: 업체 1곳, 1,980,000원 → 블루오션 후보(저경쟁)
- 주차관제장치: 업체 3곳, 20만~3,000만원 (규격별 단가 분산 큼)
- 포설형탄성포장재: 업체 2곳, 92,500~119,500원

## 마진 축 — 완성 (네이버 쇼핑 검색 API 연결됨)
- 엔드포인트: `https://openapi.naver.com/v1/search/shop.json` (헤더 인증)
- 가격 필터(액세서리·스티커·렌탈 잡음 제거)만으로 실제 제품 매칭 → 마진배수 계산 성공:
  - 심장충격기(AED): 정부 1,980,000 vs 시중 중앙값 1,410,000 → **1.40배**
  - 논슬립 매트: 정부 18,500 vs 시중 14,950 → **1.24배**
- **발견**: 단순 검색은 잡음 많음 → 품목 매칭(제조사·모델·규격 기반 + 가격필터)이 곧 우리 서비스의 AI 핵심. 데모 카테고리는 리테일 대응이 좋은 소비재(가전·사무·안전용품) 우선.

## 핵심 엔진 = 완전 검증됨 (통과)
단가(조달청 MAS) · 경쟁(업체수) · 시중가(네이버) · 마진배수 — 4요소 전부 실데이터로 연결.

## 미해결 (다음 단계, 우선순위 낮음)
- **수요 축(거래량/구매액)**: 통계 API는 최근연도 데이터 비어 불안정. 종합쇼핑몰 납품요구(거래 원장)로 확보 필요. `getDlvrReqInfoList`는 이 서비스 base에서 404 → 별도 경로 탐색 중. (당장은 경쟁+마진만으로도 핵심 데모 성립)
